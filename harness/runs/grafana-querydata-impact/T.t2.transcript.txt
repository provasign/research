1. **`backend.QueryDataHandler`** — signature `QueryData(ctx, *backend.QueryDataRequest)`. This is the plugin-SDK interface implemented by ~40 datasource backends + middleware. **It is defined in an external module dependency, so its signature cannot be "changed" from within this repo.**

2. **`clientapi.QueryDataClient`** — signature `QueryData(ctx, data.QueryDataRequest)`, **defined in this repo** at `pkg/registry/apis/query/clientapi/clientapi.go:19`. This is the datasource query-service client handler whose signature the repo owns and can change. "A field added to the query request path" maps to threading a new field through this datasource query path.

Interpretation #2 is the only coherent in-repo signature change, so I localize against `clientapi.QueryDataClient`. Its implementations and call sites are fully enumerable:
