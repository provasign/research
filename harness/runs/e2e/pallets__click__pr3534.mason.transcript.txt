### Changes Made

**File Modified**: `src/click/termui.py`

**Specific Change**: Added `pager.flush()` after each `pager.write(text)` call in the `echo_via_pager` function (line 349).

```python
# Before:
```

### Problem Description

The `echo_via_pager` function was writing content to a pager but not explicitly flushing the output buffer after each write. This caused content to be displayed only when the buffer was full or when the pager was closed, rather than being displayed immediately as it was written.

### Solution

By adding `pager.flush()` after each `pager.write(text)` call, we ensure that:
1. Content is immediately flushed to the pager
2. Users see output as it's generated rather than waiting for buffer completion
3. The behavior matches user expectations when using pagers like `less`

The fix is minimal and targeted, affecting only the specific issue without changing any other functionality. This ensures that when users call `echo_via_pager()` with content that should be displayed immediately (like help text or long outputs), it will appear in real-time rather than being delayed.

This change maintains backward compatibility while fixing the display timing issue described in the GitHub issue.
